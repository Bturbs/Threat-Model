JWT Session Token Theft at IDP Boundary ( T_07_JWT_THEFT )

Threat Status:
Unmitigated
Threat Description
During the Auth0 / Entra ID login flow (Sales Rep → IDP → Auth Token), an attacker on the same event WiFi network intercepts the OAuth2 redirect or exploits an open redirect vulnerability to steal the issued JWT. The token embeds the geo-locked residency region claim and TenantID (set at login per the Geo-routing Workflow), granting full authenticated session access identical to the victim rep — including the ability to approve and dispatch AI-generated messages to real leads.
Impact
  • Full impersonation of a legitimate sales rep: attacker accesses all event leads
  • uploads malicious data
  • and dispatches fraudulent messages to real contacts under the tenant's identity. Geo-locked JWT also binds attacker operations to the victim's approved residency region.
Security Objectives
SO-CONF SO-INT SO-ISOL
