# Threat Modeling AI Assist: Complete Guide

This guide explains the minimal files and prompts needed for AI-assisted threat modeling, and what outputs to expect.

---

## 1. Input Files

### a. AI Assist Context (YAML)
- **File:** `examples/AI assist.yaml`
- **Purpose:** Describes the system, assets, attackers, and security objectives.
- **Example:**
```yaml
meta:
  title: "Serverless File Upload Platform"
  modelId: "TM-SERVERLESS-UPLOAD"
  version: "1.0"
  scope: "Secure file upload and processing in a serverless environment"
assets:
  - ID: "A1"
    name: "API Gateway"
    type: "Service"
    description: "Entry point for file uploads"
attackers:
  - REFID: "ATT-EXT"
    name: "External Attacker"
securityObjectives:
  - ID: "SO-CONF"
    name: "Confidentiality"
```

### b. Prompt (Markdown or Text)
- **File:** `examples/ai-assist-prompt.md`
- **Purpose:** Clear instruction for the AI to generate the threat model.
- **Example:**
```
Using the provided context, generate a threat model:
- Identify relevant threats
- Describe their impact
- Propose countermeasures (no brand names or standards references)
- Include implementation steps and rationale for each countermeasure
```

---

## 2. Expected Output Files

### a. Threat Model YAML
- **File:** `threat-models/<project>/threat-model.yaml`
- **Purpose:** Full threat model generated by AI, including threats, impacts, and countermeasures.

### b. HTML Report
- **File:** `reports-output/<project>/reports/threat-model.html`
- **Purpose:** Rendered HTML report generated from the YAML file.

---

## 3. Workflow Summary
1. Provide the AI assist context YAML.
2. Supply a clear prompt (markdown or text).
3. AI generates a threat model YAML file.
4. Run the report generator to produce the HTML report.
